04

핵심 개념
Model ≠ Agent

모델과 에이전트는 독립 변수

역할SSD 비유AI 세계
기반 성능NANDModel
구동 시스템FirmwareAgent (Harness)
사용자사용 환경Pilot (사람)

NAND가 아무리 좋아도 Firmware가 못 살리면 성능이 안 나오듯
최고의 모델을 써도 에이전트가 부족하면 성능이 안 나옵니다

성능 공식

코딩 성능 = Model × Agent × Pilot // 세 변수 중 하나라도 0이면 전체가 0
Model

Opus, Sonnet, GPT-5.3, Gemini 3

Agent

Claude Code, Codex, Antigravity, Cursor

Pilot

무엇을 만들어야 하는지 정의하는 사람

기술적 가능성과
현실 제약의 분리

세 가지 영역

A — 가능 + 적용 가능

사외망 PoC에서 AI 코드 분석, 리팩토링, 테스트 생성

B — 가능하지만 제약

JIRA/Confluence 연동, 사내망 프론티어 모델

C — 기술적 한계

100% 신뢰 자율 설계, 도메인 완전 이해

"안 된다"고 판단된 항목 중 상당수가 기술적 한계(C)가 아니라 환경적 제약(B)에 해당

우리 환경에의 적용

제약 유형현재 상태해결 경로
