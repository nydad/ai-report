두 질문을 분리하기

기술적으로 가능한가 ≠ 우리 환경에서 적용 가능한가

세 가지 영역

A
기술적으로 가능 + 지금 적용 가능
예: 사외망 PoC 환경 AI 코드 분석·리팩토링·테스트 생성
B
기술적으로 가능 + 환경 제약으로 적용 불가
예: Agent의 JIRA/Confluence 연동, 사내망 프론티어 모델 직접 사용
C
기술적으로 아직 불가능
예: 100% 신뢰할 수 있는 자율 아키텍처 설계

B

실무에서 관심이 집중되는 영역

"안 된다"고 판단된 항목 중 상당수가 기술적 한계(C)가 아니라 환경적 제약(B)에 해당

제약이라면 논의의 여지가 있다 / 한계라면 기술 성숙도를 지켜보는 영역이 된다

영역 A의 가능성을 직접 확인하기 위함

기술적으로 무엇이 되는지 먼저 확인하면, 우리 환경에서 어떤 제약을 어떤 순서로 풀어갈지에 대한 논의로 자연스럽게 이어진다

조직 차원 과제

  • 내부 시스템 통합: JIRA · Confluence · Bitbucket 연동 — 기술적으론 가능 (영역 B), 보안 정책 검토 필요
  • 일하는 방법의 변화: 코드 리뷰 기준 · 품질 게이트 · 일정 산정 · SOP 전반 재검토
  • CI/CD 파이프라인 통합: AI 생성 코드의 빌드·테스트·배포 파이프라인 통합 — 파일럿 → 단계적 확대
  • 리소스와 비용: 라이선스 · 인프라 · 보안 검토 인력 · 교육 리소스 조직 차원 투자 판단
